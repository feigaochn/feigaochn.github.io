Linear Regression with Multiple Variables (Week 2)

Multiple Features

Notations:

  • \(n\): number of features
  • \(x^{(i)}\): input (features) of i-th training example
  • \(x_j^{(i)}\): value of feature j in i-th training example

Hypothesis:

  • \(x, \theta\): \((n+1)\)-dim vector (\(x_0 = 1\))
  • \(h_\theta(x) = \theta^T x\)

Gradient Descent for Multiple Variables

Matrix Dimensions:

  • \(X\): \(m \times (n+1)\): training examples stored row-wise
  • \(\theta\): \((n+1) \times 1\)
  • \(y\): \(m \times 1\)

Gradient descent:

  • Hypothesis: \(h_\theta(X) = X \theta\)
  • Parameters: \(\theta\)
  • Cost function: \(J(\theta) = \frac{1}{2m} (X \theta - y)^T (X \theta - y)\)
  • Gradient descent: \(\theta := \theta - \alpha \nabla J(\theta) = \theta - \frac{\alpha}{m} X^T (X \theta - y)\)

Feature Scaling

Idea: make sure features are on a similar scale

Mean normalization: to make features have approximately zero mean (not \(x_0\))

\(x_i := (x_i - \mu_i)/s_i\), where \(\mu_i\) is the average and \(s_i\) is range or standard deviation.

Run faster and converge in fewer iterations

Learning Rate

Make sure gradient descent is working correctly.

  • Plot \(\min_\theta J(\theta)\) vs number of iterations. It should decrease and near x axis.
  • If \(\alpha\) is too large, cost function may not decrease on every iteration, or may not converge.
  • For sufficiently small \(\alpha\), \(J(\theta)\) should decrease on every iteration.
  • But if \(\alpha\) is too small, convergence can be slow.

Try \(\alpha\) in log: ..., 0.01, 0.03, 0.1, 0.3, 1, ...

Features and Polynomial Regression

  • Assume \(h = \theta_0 + \theta_1 x + \theta_2 x^2\).
  • Just define \(x_0 = 1\), \(x_1 = x\), \(x_2 = x^2\).
  • Becomes linear: \(\sum_i \theta_i x_i\)
  • Be careful of feature scaling!

Normal Equation

Method to solve for \(\theta\) analytically.

Intuition:

Solve the equation: (partial) derivation == zero.

Example:

  • \(X\): design matrix (\(m \times (n+1)\))
  • \(y\): \(m \times 1\)-dim vector
  • \(\theta = (X^T X)^{-1} X^T y\)
  • Octave: pinv(X'*X)*X'*y

Feature scaling is NOT needed in normal equation.

Comparison:

  • Gradient Descent:
    1. need to choose \(\alpha\)
    2. many iterations
    3. work well even for large \(n\) (experience \(>10^6\))
  • Normal Equation
    1. no need
    2. no iteration
    3. computing matrix inverse is slow when \(n\) is large: \(O(n^3)\)

Normal Equation Noninvertibility (Optional)

What if \(X'X\) is not invertible? (singular/degenerate)

In Octave, pinv is pseudo-inverse and works well here.

Why?

  1. Redundant features (linearly dependent)
  2. Too many features (e.g. \(m \le n\)).
    • Delete some features, or use regularization

Vectorization

Vectorization is the process of taking code that relies on loops and converting it into matrix operations. It is more efficient, more elegant, and more concise.

%% With loops:
prediction = 0.0;
for j = 1:n+1,
  prediction += theta(j) * x(j);
end;

%% With vectorization:
prediction = theta' * x;
